# 業務フロー図作成AIアシスタント

業務改善コンサルタント向けの、**ヒアリングから図の完成まで**を支援するAIツール集です。

---

## 🎯 目的

クライアントへのヒアリング中に、その場で業務フローを可視化し、スイムレーン図として完成させる。

---

## 📦 構成

### **00. 業務改善コンサル支援Bot**
新人コンサルタント向けの**ヒアリング支援コーチ**

**使い方:**
- Google Gemini GEM または ChatGPT MyGPT に `prompt.md` の内容を登録
- ヒアリング中に会話内容を共有すると、必要な時だけアドバイス
- ロールプレイング練習にも使える

**特徴:**
- 困っている時だけ助言（基本は静かに傾聴）
- スキルレベル自動判定（インターン/ジュニア/アソシエイト）
- 「整理して」で情報を構造化

---

### **01-a. 業務ワークフロー分析（JSON出力）**
クライアントと一緒に見ながら使う**対話型分析ツール**

**使い方:**
1. Claude.ai、ChatGPT、Gemini等に `prompt.md` を読み込ませる
2. クライアントとの打ち合わせ中に画面を共有
3. AIの質問に沿ってクライアントが回答
4. その場で業務フローが構造化される
5. 最後にJSON形式で出力

**出力例:**
```json
{
  "workflow": { "name": "注文処理", ... },
  "lanes": [ {"id": "L1", "name": "営業部"}, ... ],
  "processes": [ {"id": "P1", "name": "受注"}, ... ],
  ...
}
```

---

### **01-b. スイムレーン図作成（JSON→各形式）**
JSON データを**4種類の図形式**に変換

| 形式 | 用途 | 使い方 |
|------|------|--------|
| **draw.io** | 編集しやすい図 | [draw.io](https://app.diagrams.net/) で開く |
| **PlantUML** | コードで管理 | VSCode拡張 or [PlantUML Online](http://www.plantuml.com/plantuml/uml/) |
| **Mermaid** | GitHub/Notionに埋め込み | そのまま貼り付け |
| **BPMN** | 業務プロセス標準形式 | [Camunda Modeler](https://camunda.com/download/modeler/) で開く |

**使い方:**
1. 01-aで出力されたJSONをコピー
2. 希望する形式のプロンプト（`drawio.md` 等）をAIに読み込ませる
3. JSONを貼り付け
4. 生成された図のコードをツールで開く

---

## 🚀 実際の使用手順

### **ステップ1: ヒアリング前の準備（任意）**
```
00. 業務改善コンサル支援Bot で練習
↓
ロールプレイングでヒアリング技術を磨く
```

### **ステップ2: クライアントとのヒアリング**
```
01-a. 業務ワークフロー分析 を起動
↓
画面を共有してクライアントと一緒に入力
↓
AIの質問 → クライアント回答 → その場で構造化
↓
JSON出力（コピーして保存）
```

### **ステップ3: 図の作成**
```
01-b. スイムレーン図作成 から希望の形式を選択
↓
JSONを貼り付け
↓
図のコードが生成される
↓
対応ツールで開いて完成
```

## 📊 各図形式の開き方

### **draw.io（最も編集しやすい）**
1. 生成されたXMLをコピー
2. https://app.diagrams.net/ を開く
3. 「ファイル」→「インポート」→「XMLから」
4. ペーストして「インポート」

### **PlantUML（コードで管理）**
**オンライン:**
1. 生成されたコードをコピー
2. http://www.plantuml.com/plantuml/uml/ を開く
3. ペーストして表示

**VSCode:**
1. PlantUML拡張をインストール
2. `.puml` ファイルとして保存
3. Alt+D でプレビュー

### **Mermaid（GitHub/Notion埋め込み）**
**GitHub:**
- README.md に直接貼り付け（自動表示）

**Notion:**
- コードブロック（言語: Mermaid）に貼り付け

**オンライン確認:**
- https://mermaid.live/ で表示確認

### **BPMN（業務プロセス標準）**
1. 生成されたXMLをコピー
2. https://camunda.com/download/modeler/ から Camunda Modeler をダウンロード
3. 「File」→「Open」→ XMLファイルを開く
4. 編集・実行可能

## ✨ プロジェクトの特徴

- **非エンジニアでも使える**: 専門知識不要
- **その場で完結**: ヒアリング→図完成まで1回の打ち合わせで可能
- **柔軟な出力**: 用途に応じて4形式から選択
- **AI活用**: 面倒な作業はAIに任せて、本質的な対話に集中
